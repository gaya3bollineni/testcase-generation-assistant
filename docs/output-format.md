# Generated Test Case Candidate Format

This document defines the expected structure of test-case candidates
generated by the system.

Generated test cases are proposals, not executable artifacts. Each
candidate must be easy to review, discuss, modify, or discard by a human.

## Test ID
A unique identifier for the generated test case.

Purpose:
- Enables discussion, tracking, and revision

## Description
A short, human-readable summary of the scenario being tested.

Purpose:
- Allows quick scanning and triage
- Communicates intent without reading full details

## Preconditions
Conditions that must be true before the test is executed.

Purpose:
- Makes assumptions explicit 
- Prevents hidden test dependencies

## Steps
An ordered list of actions and system responses.

Purpose:
- Describes behavior under test
- Independent of implementation details

## Expected Outcome
The observable result that indicates success or failure.

Purpose:
- Defines what correctness means for this scenario

## Risk Level
A qualitative assessment of impact:
- Low
- Medium
- High

Purpose:
- Helps prioritize review and execution
- Reflects business and operational exposure

## Rationale
An explanation of why this test case exists.

Purpose:
- Links the test to business rules, constraints, or failure modes
- Enables reviewers to judge value, not just coverage
